Python packaging

2021-12-09 | Modified: 2021-12-31 | Python ·

How can I reuse local python code in different projects?

I've collected useful utility functions to help me with this years advent of code. So far it has been great, the folder lives in the same directory as my yearly advent of code folder like this

advent-of-code/
    2020/
    2021/
        data/
        day1.py
        ...
        toolbox/
            core.py
            ...
    LICENSE.txt
    README.md

Before starting each puzzle I load up a new python file with the first line always starting with from toolbox.core import ....

Everything was great until I wanted to re-use my tool box for last years problems. I didn't want to duplicate my toolbox for every advent of code year I participate in. A logical place, I thought, would be to place the toolbox in the root of the project like so:

advent-of-code/
    2020/
    2021/
        data/
        day1.py
        ...
    toolbox/
        core.py
        ...
    LICENSE.txt
    README.md

This way I wouldn't have to duplicate my code and could expose the toolbox for each year to use. Not quite.

I first tried a relative import from ..toolbox.core import parse_input. But that didn't work.

To answer the reason why, we need to understand what happens when you run a python script.

What happens when you run a python script?

The simplest solution was to create a local package using the setuptools package that comes with a fresh installation of python 3.9.

If you want a minimal solution, look no further. Here it is:

setup.py

from setuptools import setup

setup(
    name="local_toolbox",
    version="0.1.0",
    packages=["toolbox",],
    )

And to install the package locally on the command line.

$ pip install -e .

Enjoy. :).

For those who want to know a little more, let's continue.

setup.cfg and setup.py

There are two ways to specify setup config, dynamically as parameters of the setuptools.setup() function and in a setup.cfg config file.

The equivalent setup to the first example would split across the executable and config file.

setup.py

from setuptools import setup

setup()

setup.cfg

[options]
packages = toolbox

[metadata]
name = local_toolbox
version = 0.1.0

As you can see, the name, version and packages attributes are now configured as options and metadata. There is no definitive way in using setuptools. I personally prefer to have all config in a config file. With additional optional fields such as a long_description, it is easy to edit a config file than source code.
